- **経験年数**: 約5年（インフラ構築・運用）

- **資格**: AWS認定 SAA / Developer / SysOps（2020年取得）

## 2. 経歴

### 【経験①】大手広告代理店 電車内広告配信システム（2023年8月〜2025年1月）

#### 概要
電車内広告動画配信システムのインフラ構築・運用を担当。

#### 技術スタック
- **AWS**: CloudFront, WAF, ALB, EC2, ECS, Aurora MySQL
- **コンテナ**: Docker, ECS + EC2
- **CI/CD**: GitHub Actions
- **動画処理**: Lambda + ffmpeg（4沿線対応）
- **セキュリティ**: GuardDuty, AWS Config, CloudTrail, Inspector

#### 担当業務
- CloudFront + WAF、ALB + EC2×2台によるWordPress冗長構成の設計・構築
- ECS + EC2（Docker コンテナ）基盤の構築
- Lambda + ffmpegによる動画変換パイプラインの実装
- GitHub Actionsを使用したCI/CDパイプラインの構築
- プライベートサブネット + SSM経由SSH設計
- クライアントセキュリティ基準対応、AWS Inspector脆弱性対応

#### 成果
- **コンテナ技術**: ECS + Dockerによるコンテナ基盤の設計・構築
- **CI/CD構築**: GitHub Actionsを使用した自動デプロイパイプライン構築
- **セキュリティ**: 企業セキュリティ基準に準拠したインフラ設計
- **動画処理**: サーバーレス（Lambda）を活用した動画変換パイプライン

---

### 【経験②】大手放送局グループ MSP監視運用（2024年1月〜2025年6月）

#### 概要
大手放送局グループの複数サービスに対する24/365監視運用代行を担当。

#### 技術スタック
- **AWS**: ECS Fargate, RDS, ElastiCache, ELB
- **CDN/WAF**: Fastly CDN + WAF
- **監視**: New Relic APM/Infrastructure, Zabbix
- **その他**: GCP

#### 担当業務
- New Relicによる監視設計・アラート設定
- Runbook作成（障害対応フロー）
- Fastly WAF運用（ブラックリスト管理、セキュリティレポート）
- AWS IAMポリシー設計（スイッチロール、最小権限）
- 障害一次対応フロー設計・エスカレーションルール策定
- 24/365監視運用

#### 監視対象
ECS Fargate, Fastly CDN, RDS, ElastiCache等

#### 成果
- **SRE実践**: 24/365監視運用、Runbook整備、インシデント対応フロー設計
- **監視基盤**: New Relic APM/Infrastructureを使用した監視設計
- **コンテナ運用**: ECS Fargateの運用経験
- **セキュリティ**: WAF運用、IAM最小権限設計

## 5. 強み

### 技術面
1. **AWS**: 5年間の実務経験、3つのアソシエイト資格
2. **コンテナ**: ECS Fargate/EC2 + Dockerの構築・運用経験
3. **CI/CD**: GitHub Actionsによるパイプライン構築
4. **SRE**: 24/365監視、Runbook整備、障害対応フロー設計
5. **DB**: MySQL/PostgreSQLの移行・バージョンアップ経験

### 業務面
1. **一貫した経験**: 設計→構築→運用まで一気通貫
2. **チームリード**: 設計レビュー、後輩指導
3. **大規模案件**: 放送局・新聞社・広告代理店向けシステム

### 金融案件への適性
- セキュリティを重視したインフラ設計経験
- 24/365監視運用によるミッションクリティカルなシステム運用経験
