## AIプロジェクトの本質を理解する（最重要）

### 0. AIで解決したい課題

> **商談の最初に確認すべき最重要事項**
> 先方が「なぜAIを使いたいのか」「何を解決したいのか」を理解することで、インフラ設計の方向性が決まります。

#### 基本質問
- [ ] **AIで解決したい具体的な課題は何ですか？**
- [ ] **どのような種類のデータを扱いますか？**（画像、テキスト、数値、音声）
- [ ] **AIモデルは既に存在しますか？それとも新規開発ですか？**
- [ ] **想定しているエンドユーザーは誰ですか？**（医師、患者、看護師、事務）
- [ ] **AIの出力結果をどのように活用しますか？**（診断支援、自動化、意思決定支援）

#### 詳細確認
- [ ] AIモデルの推論はリアルタイム性が必要ですか？（即時応答 vs バッチ処理）
- [ ] モデルの精度目標はありますか？（感度、特異度、F1スコア等）
- [ ] 誤判定が発生した場合のリスクと対応策は検討されていますか？

### 画像診断支援AI
医療画像から異常を検出・分類するAI

| 対象 | 具体例 | インフラ要件 |
|------|--------|-------------|
| 放射線画像 | CT/MRI/X線の異常検出、肺がん・脳腫瘍スクリーニング | GPU推論、大容量ストレージ（DICOM） |
| 病理画像 | がん細胞の検出・分類、組織診断支援 | HPC環境（大規模画像処理） |
| 眼底画像 | 糖尿病網膜症、緑内障の早期発見 | リアルタイム推論 |
| 皮膚画像 | 皮膚がん、皮膚疾患の判定 | エッジ推論（クリニック向け） |
| 内視鏡画像 | ポリープ検出、消化器がんスクリーニング | 低レイテンシ要件 |

**典型的なインフラ構成**:
- 画像保存: S3 + FSx for Lustre（高速アクセス）
- 学習: SageMaker / AWS Batch + p4d/p5インスタンス
- 推論: EKS + GPU (g5) / SageMaker Endpoints
- データ連携: DICOM対応、PACS連携

### 診断支援・臨床意思決定支援AI
患者データから診断や治療方針を提案

| 対象 | 具体例 | インフラ要件 |
|------|--------|-------------|
| 問診AI | 症状から疾患候補を提示、トリアージ支援 | NLP処理、チャットUI |
| 電子カルテ分析 | 過去データからリスク予測、再入院予測 | 構造化データ処理 |
| 治療提案 | ガイドラインに基づく治療オプション提示 | 知識ベース連携 |
| 薬物相互作用 | 処方薬の相互作用チェック | リアルタイムAPI |

**典型的なインフラ構成**:
- データ: Aurora/RDS + ElastiCache
- NLP処理: SageMaker / Bedrock（LLM活用）
- API: EKS + ALB、低レイテンシ設計
- セキュリティ: PHI対応、監査ログ必須

### 創薬・治験支援AI
新薬開発や臨床試験を効率化

| 対象 | 具体例 | インフラ要件 |
|------|--------|-------------|
| 化合物スクリーニング | 有望な化合物候補の予測 | HPC必須、大規模計算 |
| 臨床試験設計 | 最適な被験者選定、試験デザイン | データレイク、分析基盤 |
| 論文・文献解析 | 医学論文からの情報抽出 | NLP、大規模テキスト処理 |
| バイオマーカー発見 | ゲノム・プロテオミクスデータ解析 | HPC、大容量ストレージ |

**典型的なインフラ構成**:
- 計算: AWS Batch + ParallelCluster、Spotインスタンス活用
- ストレージ: S3 + FSx for Lustre（PB級データ）
- 分析: EMR / Athena / Redshift
- MLOps: Kubeflow / MLflow

### 医療事務・業務効率化AI
事務作業の自動化・効率化

| 対象 | 具体例 | インフラ要件 |
|------|--------|-------------|
| レセプトチェック | 診療報酬請求の自動チェック・修正 | ルールエンジン + ML |
| 文書作成支援 | 紹介状、診断書の自動生成 | LLM、テンプレートエンジン |
| 音声文字起こし | 診察内容の自動記録 | リアルタイム音声処理 |
| 予約最適化 | 外来予約の最適化、待ち時間予測 | 時系列予測 |

**典型的なインフラ構成**:
- API: ECS/EKS + Fargate（コスト効率重視）
- LLM: Amazon Bedrock / SageMaker
- データ: Aurora Serverless（可変負荷対応）
- 連携: 電子カルテAPI、レセコン連携
